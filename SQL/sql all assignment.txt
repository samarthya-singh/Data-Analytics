8).Explain Normalization in the Context of Databases

Normalization is the process of organizing data in a database to reduce redundancy (duplicate data) and improve data integrity.
It divides large tables into smaller, related tables and defines relationships between them.

a.First Normal Form (1NF)

No repeating groups

Each column contains atomic (single) values

b. Second Normal Form (2NF)

Must be in 1NF

No partial dependency on part of a composite key

c. Third Normal Form (3NF)

Must be in 2NF

No transitive dependencies (non-key column should not depend on another non-key column)

10) What does a JOIN in MySQL mean?

A JOIN in MySQL is used to combine rows from two or more tables based on a related column between them.

It allows you to retrieve data that is spread across multiple tables by matching rows using a common field (usually a primary key and a foreign key).

12.)What is the role of the MySQL JOIN clause in a query, and what are some common types of joins?
Role of the MySQL JOIN clause:

The JOIN clause is used to combine rows from two or more tables based on a related column between them.
It allows you to retrieve data that is stored across multiple tables by linking them through keys (usually primary key and foreign key).

In short:
 JOIN helps you fetch connected data from multiple tables in a single query.

Common Types of Joins in MySQL
1INNER JOIN

Returns only the rows where both tables have matching values.

SELECT * 
FROM A 
INNER JOIN B ON A.id = B.id;

2 LEFT JOIN (LEFT OUTER JOIN)

Returns all rows from the left table and only matching rows from the right table.

3 RIGHT JOIN (RIGHT OUTER JOIN)

Returns all rows from the right table and only matching rows from the left table.

4 FULL JOIN (FULL OUTER JOIN)

MySQL does not support FULL JOIN directly, but it can be simulated using UNION of LEFT + RIGHT join results.

5 CROSS JOIN

Returns the Cartesian product‚Äîevery row of table A combined with every row of table B.
